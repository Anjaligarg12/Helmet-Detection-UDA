# Unsupervised Domain Adaptation (UDA) for Helmet Detection

This repository contains a full edge-to-edge working prototype for adapting a helmet detection model trained on a source domain (e.g., India) to a target domain without labels (e.g., Southeast Asia).

## Architecture Approach

The pipeline strictly utilizes the following:
1. **CycleGAN (PyTorch)**: A lightweight ResNet-based GAN implementation to perform unpaired image-to-image translation. Source domain images (India) are adapted into the target domain style (SEA) while preserving the semantic content (helmets).
2. **YOLO (Ultralytics YOLOv8)**: A fast and lightweight detection model trained on both the original baseline and the CycleGAN-adapted datasets to prove out the efficacy of Unsupervised Domain Adaptation.

## Project Structure

```text
│
├── data/                    # Generated via utils/preprocess.py
│   ├── source_india/        # (Domain A) With Helmet Bounding Boxes

├── detection/               
│   ├── train_yolo.py        # Logic to train baseline and adapted YOLOs
│   ├── evaluate.py          # Metric evaluations (mAP, Precision, Recall)
│   └── {dynamic}.yaml       # Yaml configurations for YOLO datasets
│
├── utils/                   
│   └── preprocess.py        # Bootstrapping script to generate Synthetic dummy data
│
├── outputs/                 
│   ├── samples/             # Visual outputs of GAN translation
│   ├── metrics/             # Evaluation scores (results.json)
│   └── (yolo weights)       # Stored optimal weights for YOLOv8 nano

```

## How To Run

1. Create a fresh Python 3.10+ Virtual Environment.
2. Install the lightweight dependencies:
```bash
pip install torch torchvision ultralytics opencv-python pyyaml
```
3. Execute the pipeline:
```bash
python main.py
```

### What `main.py` Does
- **Step 1:** Automatically populates `data/` with *dummy synthetic* data of "helmets" formatted exactly for UDA and YOLO. (You can immediately replace the contents of these folders with your real dataset).
- **Step 2:** Trains a lightweight CycleGAN for 2 epochs on `data/source_india` and `data/target_sea`.
- **Step 3:** Restyles all images in `source_india` and saves them (along with labels) to `data/adapted/`.
- **Step 4:** Trains Baseline `YOLOv8n` on original India-style data.
- **Step 5:** Trains Adapted `YOLOv8n` on SEA-style adapted data.
- **Step 6:** Evaluates both on `data/test_sea` and reports `mAP@50`, proving the UDA performance boost.

## Reviewing Outputs
After execution, results will be available in the `outputs/` folder.
* Open `outputs/metrics/results.json` to view the quantitative gains from UDA.
* See `outputs/samples/` for a visual comparison of Domain Translation performance.
